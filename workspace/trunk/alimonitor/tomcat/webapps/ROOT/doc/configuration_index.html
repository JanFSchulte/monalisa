If a detector was not included in the data taking, it's status is not defined in the table.

The numerical fields (including detectors' status) can be filtered using the follwing syntax:

  • "*" - any defined value
  • "!" - is not defined
  • "123" - a numerical exact value
  • "123,200,300,400" - a list
  • "123:1000" - a range
  • "123:" or ":500" - open-ended range (<=123 respectively >=500)
  • "!123" - excluding an element
  • "!123:200" - excluding a range
  • "123,200:500,!250,!300:400,1000:" - putting everything together

Filtering the string fields is similar, for example:

  • "*" -any defined value
  • "!" - is not defined, field must be empty
  • "lumi" - search for this substring in the field, case insensitive
  • "!lumi" - this substring must not exist
  • "!lumi,!ssd" - skip if any of the two values exists
  • "lumi,ssd" - any of the two
  • "lumi,!ssd" - includes first, but not the second
